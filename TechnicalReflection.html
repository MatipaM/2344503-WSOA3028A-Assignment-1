Matipa's Portfolio Website

Technical Reflection

I made the navigation bar position fixed so that no matter how far the user scrolls down on the page, they can use the navigation bar immediately as it will always be positioned at the top of the screen. I also made sure to only use one script to control where the navigation bar was positioned to prevent it from moving around.

I decided to add text to the bottom of my icons, as I received feedback that my icons were not distinctive enough to be used without text. I managed to scale this efficiently for a smaller device and was happy with how the navigation bar looked on all three devices.

I managed to figure out how to position the carousels on the app dev page (Firtech.html) side by side by using grids, which kept the carousels in place when resizing. I changed the carousels to be placed one after another on a mobile device so that the user would have enough space to be able to view each carousel clearly. I also used a double column to display the logos and text below the carousels. As I thought a single column would make the text confusing, as the user may not know if the paragraph is talking about the loo above or below.

I created a 2x3 grid for the blog posts page and then a 1x3 grid for the reflections and development page. I found that using 3 column cards gave the website a more uniform look. It also allowed all the blog posts to be visible on a single page so that the user would not have to scroll down to see all the reflections. All the grids collapse into a single column on a mobile screen. This is to compensate for the smaller fonts and images. As a grid would be much more difficult to see on a 500pixel device or less.

When it came to semantic markup, I made an effort to mark all elements on my page with tags that I thought were most appropriate. Initially, I wasn't too sure what content to place in the header and footer section of my website. After doing some research I decided to add references/citations of pixel art from other artists which I had added to my website as well the contact details I was comfortable with recruiters seeing. In the header of each webpage, I placed the navigation bar.

I used an external API to make my Contact Me page functional. The contact me page, validates the users email and password length with html. I am then forwarded the email the user has typed in the message box. The page redirects to a Thank you page I created, where the user can use the navigation bar to visit any other pages they find interesting.
